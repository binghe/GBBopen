load-blackboard-repository

load-blackboard-repository   pathname &key confirm-if-not-empty disable-events retain-classes retain-event-functions retain-event-printing => pathname[Function]

Purpose
Load the blackboard repository (all unit instances and space instances) that has been saved to a file previously by save-blackboard-repository.

Package   :gbbopen

Module   :gbbopen-core

Arguments

pathname     A pathname designator
confirm-if-not-empty     A generalized boolean indicating whether the user must confirm loading when the current blackboard repository contains unit instances (default is true)
disable-events     A generalized boolean (default is t)
retain-classes     An extended unit-classes specification (see below)
retain-event-functions     A generalized boolean (default is nil)
retain-event-printing     A generalized boolean (default is nil)

Returns
The pathname of the saved blackboard-repository data file or nil if replacing a non-empty blackboard repository is not confirmed.

Events
If disable-events is nil, the following events may be signaled as unit instances and space instances are deleted prior to loading:

  • unlink-event
  • remove-instance-from-space-instance-event
  • delete-instance-event
  • instance-deleted-event

Description
If pathname does not specify a file type, the type bb is added to it. Then, *default-pathname-defaults* is used to supply any missing components to pathname.

See also
    omitted-slots-for-saving/sending
    save-blackboard-repository

Examples
Load the GBBopen Tutorial Example application (without running it) and then load a blackboard repository that was saved previously: 

  > :tutorial-example :noautorun 
  > (load-blackboard-repository "tutorial")
  #P"/Users/corkill/tutorial.bb"

Try loading the repository again: 

  > (load-blackboard-repository "tutorial")
  The blackboard repository is not empty.
  Continue anyway (the current contents will be deleted)? no
  nil
