Portable Threads

The :portable-threads module provides a uniform interface to commonly used multiprocessing entitites. Where possible, these entities do something reasonable in Common Lisp implementations that do not provide multiprocessing; entities that make no sense without multiprocessing signal errors (as noted with each entity).

GBBopen adds the feature :multiprocessing-not-available on Common Lisp implementations that do not provide multiprocessing support. The feature :thread-scheduling-not-available is added on implementations that provide multiprocessing, but not controllable thread scheduling.
